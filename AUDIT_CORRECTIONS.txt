#
# 1. PROCEDURE: Lire UNE image de carte -> comparer au JSON -> noter dans ce fichier -> corriger le JSON si erreur -> passer a la suivante
# 2. JAMAIS lire plusieurs images en meme temps (risque de crash)
# 3. SYMBOLES SUR LES CARTES:
#    - [↯] = effet INSTANTANE (se declenche a la pose, une seule fois)
#    - [⧗] = effet CONTINU (reste actif tant que le personnage est face visible)
#    - Ces symboles peuvent apparaitre dans TOUT type d'effet (MAIN, AMBUSH, UPGRADE, SCORE)
#    - NE PAS retirer ces symboles des descriptions, ils sont corrects et importants
#
# 4. CE QU'IL FAUT VERIFIER SUR CHAQUE CARTE (dans l'image):
#    a) Nom du personnage (name_fr / name_en)
#    b) Titre de la carte (title_fr)
#    c) Cout en chakra (nombre en haut a gauche)
#    d) Puissance (nombre en bas a droite)
#    e) Keywords (badges en bas a gauche de la carte, ex: "Team 7", "Sannin", "Sound Ninja")
#       ATTENTION: le titre n'est PAS un keyword! Les keywords sont dans des badges specifiques
#    f) Group/Village (icone en haut a droite: Leaf Village, Sand Village, Sound Village, Akatsuki, Independent)
#    g) Type et texte des effets (MAIN, AMBUSH, UPGRADE, SCORE)
#    h) Rarete (bordure/couleur de la carte)
#
# 5. ERREURS CONNUES FREQUENTES DANS LE JSON:
#    - "Pouvoir" comme keyword -> c'est FAUX, verifier sur l'image
#    - "Team Dosu" comme keyword -> c'est FAUX, sur les cartes c'est "Sound Ninja"
#    - "Academy" comme keyword -> c'est FAUX, c'est souvent le titre pas un keyword
#    - Group "Independent" sur des cartes Leaf Village UC -> verifier sur l'image
#
# 6. FORMAT DES CORRECTIONS:
#    [ID] NOM - OK (si tout est correct)
#    [ID] NOM - ERREUR
#      champ: VALEUR_JSON -> VALEUR_CORRECTE
#
# 7. FICHIER JSON A MODIFIER: lib/data/card-data.json
#
# 8. POUR REPRENDRE: chercher "--- AUDIT EN COURS" en bas du fichier pour savoir ou on en est
#
# =======================================================
